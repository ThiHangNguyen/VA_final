Détecte les coins d’une feuille A4 dans une image couleur.

Détecte les 4 coins d'une feuille A4 (ou forme équivalente) dans une image BGR.

Cette fonction applique une série d'étapes robustes :

  • conversion en niveaux de gris,
  • floutage pour réduire le bruit,
  • seuillage dynamique basé sur la luminance moyenne d'une ROI centrale,
  • détection des contours,
  • approximation polygonale pour vérifier qu’on a bien un quadrilatère.

Si un contour quadrilatéral est trouvé, ses 4 coins sont ordonnés (TL, TR, BR, BL) et retournés dans imagePts.

Parameters
frameBGRImage couleur d’entrée (BGR).
imagePtsVecteur de 4 points 2D (float) retourné si détection réussie.
Returns
true si une feuille A4 (ou une forme assimilée) est détectée.

Pipeline de détection :

  • Conversion en niveaux de gris + floutage
  • Seuillage dynamique basé sur la moyenne d'une ROI centrale
  • Recherche de contours
  • Sélection du plus grand quadrilatère
  • Ordonnancement TL/TR/BR/BL
Parameters
frameBGRImage d’entrée en couleur (BGR).
[out]imagePtsVecteur de 4 points image (ordonnés).
Returns
true si la détection est réussie.

Dessine les coins détectés avec des couleurs et étiquettes.

Dessine les 4 coins ordonnés sur une image avec des couleurs et labels.

  • TL (rouge)
  • BL (jaune)
  • BR (bleu)
  • TR (vert)
Parameters
imgImage sur laquelle dessiner.
ptsPoints dans l'ordre TL, BL, BR, TR.

Les coins sont étiquetés "TL", "TR", "BR", "BL" dans l'ordre. Permet de vérifier visuellement que l’ordre des points est correct.

Parameters
imgImage sur laquelle dessiner les coins.
ptsPoints 2D ordonnés (taille 4, en float).

Trie et ordonne les 4 coins détectés dans l'ordre TL, BL, BR, TR.

Ordonne 4 points dans l’ordre (TL, TR, BR, BL) selon leur position spatiale.

Parameters
approxVecteur contenant 4 points détectés (désordonnés).
orderedVecteur de sortie contenant les 4 coins dans l'ordre TL, BL, BR, TR.
Returns
true si le tri a été effectué correctement.

Ce tri est basé sur la position verticale (y) en priorité, puis sur la distance par rapport au coin supérieur gauche. Il est utile pour garantir la cohérence géométrique en AR et en solvePnP.

Parameters
approx4 points issus d’une approximation polygonale (e.g., approxPolyDP).
[out]orderedPoints convertis en float et ordonnés (taille = 4).
Returns
true si succès (approx contient bien 4 points).
